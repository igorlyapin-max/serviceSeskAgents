# Этап 13.31: создание ReAct-вызова из привязки операций

## Цель

Убрать ручное копирование одноименных параметров между endpoint-операцией, `ReAct-вызовом ИИ` и `Привязкой операций`.

## Решение

В разделе `Привязка операций` добавлен режим `Создать и привязать ReAct-вызов ИИ`.

Администратор выбирает:

- подключение;
- endpoint-операцию;
- техническое имя нового ReAct-вызова;
- тип действия;
- описание;
- видимые для ReAct входные параметры;
- поля ответа, доступные ReAct.

После сохранения UI создает новый объект `tools` и сразу записывает в него:

- `parameters_schema`;
- `result_schema`;
- `endpoint_bindings`;
- `parameter_mapping`;
- `result_mapping`;
- версию и статус контракта из endpoint-операции.

## Правила

- Имя нового ReAct-вызова по умолчанию берется из `operation_id`.
- Если такое имя уже занято, UI предлагает имя с суффиксом.
- Для read-only вызова политика по умолчанию: низкий риск, без подсказки согласования, автоисполнение допустимо.
- Для action-вызова политика по умолчанию: средний риск, требуется подсказка согласования, автоисполнение не включается.
- После создания UI переключается в режим `Привязать существующий` на созданный ReAct-вызов.

## Критерии выхода

- Администратор может создать ReAct-вызов из endpoint-операции без перехода в отдельный раздел `ReAct-вызовы ИИ`.
- Повторяющиеся одноименные поля endpoint и ReAct заполняются автоматически.
- Существующий режим привязки уже созданного ReAct-вызова сохранен.
