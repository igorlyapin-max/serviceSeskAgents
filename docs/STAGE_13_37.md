# Этап 13.37. Компактный UX обогащения контекста

## Цель

Применить общий UX-подход из `docs/UI_UX_APPROACH.md` к блоку `1. Разрешение атрибутов -> Обогащение контекста`.

Администратор должен видеть цепочку ReAct-вызовов как компактный сценарий, а не как набор больших раскрытых форм.

## Что изменено

- Верхний уровень блока `Обогащение контекста` представлен компактной таблицей шагов.
- В таблице показываются номер, название, ReAct-вызов, сущность результата, действие при ошибке и действия.
- Одновременно раскрыт только один редактор шага.
- Новый шаг раскрывается сразу после добавления.
- Параметры ReAct-вызова отображаются плотной таблицей.
- Источники параметров показываются человекочитаемо: слот, выходной слот, сущность предыдущего шага, константа, секрет или техническая ссылка.
- Контракт результата свернут по умолчанию в строку `Контракт результата: N полей`.
- При смене ReAct-вызова UI пересобирает параметры и поля результата по контракту выбранного вызова.

## Контракт

Backend-контракт не менялся. UI продолжает сохранять тот же `enrichment_steps`.
