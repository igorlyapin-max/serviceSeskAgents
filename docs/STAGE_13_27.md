# Этап 13.27. Уточнение у клиента и эскалация оператору

## Цель

Развести два разных процесса:

- `Уточнение у клиента` - AI задает вопрос, ждет ответ клиента и продолжает тот же сценарий.
- `Эскалация оператору` - AI прекращает самостоятельную обработку и передает оператору пакет контекста.

## Что изменено

- Dry-run возвращает явные поля:
  - `next_client_question`;
  - `client_question`;
  - `awaiting_client_response`;
  - `operator_escalation`;
  - `escalation_package`.
- В Admin UI канал взаимодействия визуально разделен на:
  - `Уточнения у клиента`;
  - `Эскалация оператору`.
- В профилях разрешения атрибутов уточняемые слоты относятся к клиентскому вопросу, а handoff package называется пакетом эскалации оператору.
- В Operator UI вопрос по слоту отображается как клиентское уточнение, а шаг решения отдельно показывает ожидание клиента и передачу оператору.
- Read-only граф оркестрации получил отдельные ветки ожидания клиента и эскалации оператору.

## Сохраненные алиасы

Для стабильности текущих интеграций dry-run пока оставляет старые технические поля:

- `next_question`;
- `question_delivery`;
- `escalation_action`.

Новые интерфейсы должны использовать новые семантические поля.
