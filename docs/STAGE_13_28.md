# Этап 13.28. Очистка активных вариантов меню администратора

## Что изменено

- В `1. Разрешение атрибутов` поле `Тип выполнения` переименовано в `Источник данных для разрешения`.
- Активный выбор источника ограничен `endpoint-операцией чтения` и `отключено`.
- Для отключенного источника скрываются выбор ReAct-вызова, маппинг параметров и описание результата операции.
- В `0. Слоты` поле `Способ заполнения` переименовано в `Как получить значение слота`.
- В `4. ReAct-вызовы и матрица запуска` поле `Вид запуска` переименовано в `Режим исполнения сейчас`.
- В матрице запуска больше не показывается отдельный целевой режим и не предлагается `согласование руководителя`.
- В `Интеграциях` активными adapter types оставлены только `mock` и `n8n webhook`.
- В `Каналах взаимодействия` действия канала фильтруются по режиму канала, технические поля ReAct-вызова/endpoint/operation скрыты.

## Backend validation

- Активный профиль разрешения атрибута может использовать только `react_call` или `disabled`.
- Включенный endpoint может использовать только `mock` или `n8n_webhook`.
- `approver_approval` запрещен до реализации отдельного approver flow.
- `execution_level` и `target_execution_level` должны совпадать.
- Действия канала должны соответствовать режиму канала.

## Конфигурация

- `queue.known_incidents` отключен в дефолтном каталоге интеграций.
- Активная конфигурация интеграций обновлена: неисполняемые adapters отключены.
- Активная матрица ReAct-вызовов обновлена: текущий и целевой режим запуска синхронизированы.
