# Этап 13.26. Read-only граф оркестрации

## Цель

Администратор получает визуальную карту оркестратора без права редактировать сам граф. Граф показывает, как текущие сущности конфигурации складываются в пять шагов обработки обращения и связанные технические блоки.

## Что изменено

- Добавлен API `GET /admin/orchestration-graph`.
- Поддержаны режимы:
  - `view=base` для базовой карты оркестрации;
  - `view=scenario&scenario_id=<id>` для карты конкретного сценария.
- Ответ API содержит read-only узлы, ребра, layout и предупреждения конфигурации.
- В Admin UI добавлен раздел `Сценарии обработки -> Граф оркестрации`.
- Граф рендерится как SVG.
- Поддержаны масштабирование `+`, `-`, `100%`, `Вписать`, перетаскивание полотна и zoom колесом мыши.
- Клик по узлу открывает панель деталей и ссылки в существующие разделы настройки.

## Ограничения

- Узлы и ребра нельзя создавать, удалять или редактировать из графа.
- Граф не является новым источником истины.
- Фактические прогоны по `execution_trace` пока не накладываются на граф; это следующий возможный этап.
