# Этап 13.39. Обязательность поля результата автозаполнения

## Цель

Разделить два уровня обязательности:

- контракт ReAct-вызова описывает, какие поля endpoint может вернуть;
- профиль автозаполнения описывает, какие поля результата нужны конкретному сценарию.

## Что сделано

- В UI поле `required_for_success` подписано как `Поле результата обязательно для профиля`.
- Флаг можно включить для любого поля результата, даже если оно не входит в `required` общего контракта ReAct-вызова.
- Backend dry-run возвращает статус `missing_required_result_field`, если обязательное для профиля поле результата не пришло.
- В ответ dry-run добавлены:
  - `missing_required_result_fields`;
  - `missing_required_slots`.
- Operator UI показывает отсутствующие обязательные поля результата в таблице автозаполнения.

## Правило

Если поле результата обязательно для профиля и не вернулось, профиль не считается успешным. Найденные необязательные значения могут остаться в `output_values`, но сценарий продолжает обработку отсутствующих обязательных слотов обычным путем: вопрос клиенту, разрешение атрибута или эскалация оператору.
