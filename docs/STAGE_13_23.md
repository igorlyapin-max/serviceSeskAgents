# Этап 13.23: вкладочная панель оператора

Этап 13.23 разгружает операторский интерфейс: слева остается контекст работы с заявкой, справа оператор переключается между тематическими вкладками.

## Что изменено

- Основной экран `/operator` переведен на двухколоночный layout.
- Левая колонка содержит текст заявки, выбор сценария, режим тестового прогона, вопросы по слотам и введенные ответы.
- Правая область содержит вкладки:
  - `5 шагов`;
  - `Результат`;
  - `Кейс`;
  - `Согласования`;
  - `База знаний`;
  - `Трассировка`.
- Вкладка `Трассировка` сохраняет внутренние режимы `RAG`, `Инструменты` и `Диагностика`.
- На экранах меньше 1200 px интерфейс переходит в одну колонку, а вкладки остаются горизонтально доступными.
- Сценарная работа не запускается автоматически при открытии страницы, вводе текста или смене настроек.
- Кнопка `Анализировать` фиксирует текст заявки в текущей операторской итерации, и уточнения слотов используют этот сохраненный текст.

## Техническое решение

- Разметка остается статической в `apps/operator-ui/static/index.html`.
- Клиентское состояние `activeMainTab` управляет видимостью блоков через `data-main-tab` и `data-main-panel`.
- Клиентское состояние `workflowStarted` отделяет подготовку экрана от запуска сценарной работы.
- Клиентское состояние `ticketTextSnapshot` хранит текст заявки, сохраненный кнопкой `Анализировать`.
- Существующие API и render-функции не менялись.
- Текст `JSON` убран из пользовательского интерфейса и заменен на `Диагностика`.
