# Этап 13.36. Последовательное обогащение контекста

## Цель

Упростить `1. Разрешение атрибутов`: вместо одной технической операции профиль теперь содержит последовательный список ReAct-вызовов. Каждый шаг сохраняет результат как именованную сущность, которую можно использовать в следующих шагах и в LLM-правиле.

## Что изменено

- В контракте `attribute_resolution_profiles` основное поле теперь `enrichment_steps`.
- В каждом шаге задаются:
  - название шага;
  - ReAct-вызов;
  - маппинг параметров из `slot:`, `output:`, `entity:`, `constant:` или `secret:`;
  - имя сущности результата;
  - поля результата сущности;
  - действие при ошибке.
- Backend выполняет шаги сверху вниз и возвращает `enrichment_entities` в состоянии разрешения.
- Валидация запрещает ссылки на будущие сущности и проверяет существование ReAct-вызова.
- Admin UI показывает блок `Обогащение контекста`.
- Operator UI показывает не одиночную операцию, а список шагов обогащения.
