# Панель оператора

Панель оператора отдается FastAPI на `/operator` из `apps/operator-ui/static`.

Текущая модель работы:

- Двухколоночная рабочая область: слева контекст заявки и уточнения, справа вкладки деталей обработки.
- Ручной ввод текста заявки в одно неструктурированное поле.
- Выбор сценария обработки обращения из сценарной модели Config Registry.
- Сценарная работа начинается только после нажатия `Анализировать`.
- Кнопка `Анализировать` фиксирует текущий текст заявки; следующие итерации обогащения используют этот сохраненный текст.
- Вкладка `5 шагов` сначала показывает `Трассу тестового прогона`: что вызывалось, с какими параметрами, какой результат вернулся и какие слоты были заполнены из каких полей ответа.
- Пять шагов оркестратора на вкладке свернуты по умолчанию и раскрываются по маркеру `>`:
  - прием и нормализация;
  - классификация и маршрутизация;
  - ReAct-планирование;
  - выполнение и инструменты;
  - решение и эскалация.
- Вкладки `Результат`, `Кейс`, `Согласования`, `База знаний` и `Трассировка` для переключения между крупными блоками без сжатия экрана.
- Обогащение заявки вопросами по одному недостающему слоту за раз.
- Отображение mini-workflow разрешения слота: текущий шаг, выполненные попытки, причина уточнения и пакет передачи человеку.
- Отображение групп действий ReAct и стоп-условий из блока `3. ReAct-планирование`.
- Отображение канала взаимодействия сценария: доставка вопроса, таймаут ожидания, действие при отсутствии ответа и действие эскалации.
- Отображение профилей действий канала для обычной передачи, отсутствия ответа, Major Incident и policy blocked.
- Отображение условий передачи и состава пакета передачи из блока `5. Решение и эскалация`.
- Показ готовности ReAct-вызовов по матрице запуска, привязке к слотам, параметрам вызова и единому виду запуска.
- Отображение AI-решения.
- Отображение уточнения и эскалации.
- Согласование или отклонение action tools оператором.
- Отображение результата интеграции или ранбука.
- Копируемый текст для ручного создания заявки в ServiceDesk.
- Трассировка найденных RAG-фрагментов и вызовов инструментов.
- Перестроение базы знаний по явному действию оператора.
- Элементы управления обратной связью: верно, неверно, исправлено.
- Статус кейса и последние события timeline.

Операторские сценарные API:

- `GET /operator/scenarios`.
- `GET /operator/scenarios/{scenario_id}`.
- `POST /operator/scenarios/{scenario_id}/simulate`.

Эти endpoint'ы используют тот же сценарный источник истины, что и консоль администратора, но доступны через операторское право `cases.operate`.

Запустить проверки:

```bash
PYTHON=.venv/bin/python make stage8-check
```

Запустить smoke-проверку:

```bash
./scripts/stage8-smoke.sh
```

```bash

./scripts/stage10-smoke.sh
```
