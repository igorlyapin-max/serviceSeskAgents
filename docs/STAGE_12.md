# Этап 12: MVP консоли администратора

Этап 12 добавляет отдельную административную консоль на `/admin`. Консоль работает поверх Admin API и не используется для обработки обращений пользователей.

## Реализовано

- Статический интерфейс администратора, обслуживаемый backend:
  - `GET /admin`.
  - `GET /admin/static/app.js`.
  - `GET /admin/static/styles.css`.
- Разделы интерфейса:
  - Панель обзора.
  - База знаний.
  - Сценарии обработки.
  - Вызовы и интеграции:
    - Интеграции.
    - ReAct-вызовы ИИ.
    - Привязка операций.
  - Рабочий процесс.
  - Модели.
  - Контроль качества.
  - Аудит.
  - Безопасность.
- Выбор инициатора разработки через `X-ServiceDesk-Actor`.
- Левое меню администратора, верхняя панель и строка статуса операций закреплены при прокрутке длинных форм.
- Безопасные действия из UI:
  - перестроение базы знаний;
  - тестовый поиск;
  - перенос обратной связи в оценочные кейсы;
  - запуск оценки.
- Представления только для чтения:
  - каталог инструментов;
  - каталог точек интеграции;
  - состояния рабочего процесса и правила переходов;
  - каталог промптов;
  - конфигурация моделей;
  - пользователи, роли и права;
  - ссылки на секреты;
  - события аудита.

## Новые Admin API

- `GET /admin/prompts/catalog`.
- `GET /admin/feedback`.
- `GET /admin/evaluations/runs`.

## Архитектурные решения

- Интерфейс администратора обращается только к HTTP Admin API.
- Интерфейс администратора не читает хранилище напрямую.
- Операторский интерфейс и интерфейс администратора разделены маршрутами:
  - `/operator`.
  - `/admin`.
- Опасные действия остаются за permission checks этапа 11.
- Production-конфигурация пока не редактируется: процесс `draft/validate/activate` остается для этапа 13.
- MVP интерфейса реализован как статический интерфейс, обслуживаемый backend, без npm-зависимостей, чтобы не блокировать локальную разработку установкой frontend-инструментов. Целевой стек React/TypeScript/Vite/Ant Design остается направлением миграции поверх тех же Admin API.

## URL

- Интерфейс администратора: `http://127.0.0.1:18088/admin`.
- Операторский интерфейс: `http://127.0.0.1:18088/operator`.

## Ограничения этапа

- Управление промптами, инструментами, рабочим процессом и моделями пока доступно только для чтения.
- Управление пользователями и ролями пока доступно только для чтения из каталога безопасности.
- Управление workflow n8n отображается через каталог точек интеграции, без restart/cancel.
- Полноценная миграция на React, активация конфигурации, регрессионные проверки и Vault/IP/MFA остаются для следующих этапов.
