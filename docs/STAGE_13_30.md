# Этап 13.30: видимый контракт ReAct поверх endpoint-операции

## Цель

Убрать смешение слотов сценария и технического маппинга endpoint.

Раздел `Привязка операций` больше не выбирает слоты. Он показывает контракт выбранной endpoint-операции и задает, какие входные параметры и поля ответа становятся видимыми для ReAct-вызова.

## Модель

- `Endpoint-операция` хранит технический контракт интеграции: `request_schema` и `response_schema`.
- `ReAct-вызов ИИ` хранит видимый контракт оркестратора: `parameters_schema` и `result_schema`.
- `Привязка операций` строит границу между ними:
  - `endpoint request parameter -> react parameter / constant / secret`;
  - `react result field -> endpoint response field`.

Слоты сценария участвуют позже: в матрице запуска или профиле разрешения атрибута они заполняют уже готовые параметры ReAct-вызова.

## UI

В `Привязка операций` блок `Маппинг payload операции` заменен на `Входные параметры endpoint`.

Для каждого параметра endpoint показываются:

- имя и тип параметра endpoint;
- обязательность для endpoint;
- источник значения: `параметр ReAct-вызова`, `константа`, `секрет`, `не использовать`;
- имя параметра ReAct, если источник `параметр ReAct-вызова`;
- обязательность параметра для ReAct.

Блок `Маппинг результата операции` заменен на `Поля ответа, доступные ReAct`.

Для каждого поля ответа endpoint показываются:

- имя и тип поля endpoint-ответа;
- признак `Вернуть в ReAct`;
- имя поля результата ReAct;
- обязательность поля результата ReAct.

## Сохранение

При сохранении `Привязать` UI пересобирает:

- `tool.parameters_schema`;
- `tool.result_schema`;
- `binding.parameter_mapping`;
- `binding.result_mapping`.

Переключатель `Синхронизировать контракт ReAct` удален: видимый контракт теперь редактируется прямо в строках endpoint-параметров и endpoint-ответа.

## Правила

- Обязательный параметр endpoint должен иметь источник значения.
- Параметры с источником `константа` или `секрет` не попадают в `parameters_schema` ReAct.
- Необязательный параметр endpoint можно не использовать.
- В ReAct-результат можно вернуть любой набор полей endpoint-ответа.
- Имена полей ReAct могут отличаться от технических имен endpoint.

## Критерии выхода

- В `Привязка операций` нет выбора слотов сценария.
- Администратор видит все параметры endpoint-операции, а не только обязательные.
- Администратор может открыть ReAct только нужный набор входных параметров и полей ответа.
- Smoke-проверки фиксируют новую терминологию UI.
