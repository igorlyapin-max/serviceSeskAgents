# Этап 13.22: расширенная классификация маршрутов

Этап заменяет плоские ключевые слова на правила классификации. Обратная совместимость для `keywords` и `negative_keywords` не поддерживается.

## Реализовано

- В `classification_routes` используется `rules.rule_items`.
- Каждое правило задает:
  - текст признака;
  - тип совпадения: `phrase`, `word`, `contains`;
  - влияние: `positive` или `negative`;
  - вес;
  - обязательность;
  - блокировку;
  - пояснение для оператора.
- Dry-run оценивает все активные маршруты и возвращает:
  - выбранный маршрут;
  - confidence;
  - уровень решения;
  - совпавшие позитивные и негативные признаки;
  - top-N альтернатив.
- Admin UI редактирует правила карточками.
- Operator UI показывает причины классификации в шаге `2. Классификация и маршрутизация`.
