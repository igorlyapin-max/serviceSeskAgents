# Этап 13.21: интерпретация результата операции в профилях разрешения атрибутов

Этап убирает неоднозначность в `1. Разрешение атрибутов`: входные параметры операции и поля ответа операции настраиваются отдельно.

## Реализовано

- `candidate_mapping` заменен на `result_policy`.
- Матрица решений использует случаи:
  - `empty_result`;
  - `single_result`;
  - `multiple_results`;
  - `source_error`;
  - `attempt_limit`.
- `result_policy` поддерживает два типа ответа:
  - `list` - массив результатов по `list_path`;
  - `object` - один объект по `object_path` и/или boolean-признак `success_path`.
- Входные параметры операции можно заполнять из признаков профиля, слотов сценария, данных обращения, контекста, констант и секретов.
- UI показывает:
  - настройки операции были историческим промежуточным шагом; текущая целевая модель использует `Обогащение контекста`;
  - `Входной параметр операции`;
  - `Значение взять из`;
  - `Как оценивать результат операции`.
